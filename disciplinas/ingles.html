Conteúdo da Disciplina

Ementa

Estudo dos princípios de bancos de dados relacionais. Criação e manipulação de estruturas de dados utilizando SQL. Desenvolvimento de procedimentos armazenados (procedures), funções definidas pelo usuário (functions), visualizações (views) e controle de acesso (DCL). Aplicação de boas práticas no uso de comandos SQL para garantir integridade, segurança e eficiência no acesso aos dados.

Tópicos Abordados

  • Conceitos de banco de dados relacional
  • Comandos SQL básicos (DDL e DML)
  • Comandos de controle de acesso com DCL (GRANT e REVOKE)
  • Criação e uso de views
  • Criação e execução de stored procedures
  • Criação e uso de funções definidas pelo usuário
  • Diferenças entre procedures e funções
  • Uso de funções em consultas SQL
  • Boas práticas de organização e segurança no banco de dados

Metodologia

A disciplina é conduzida de forma prática e expositiva, com aulas que combinam teoria e exercícios diretamente aplicados em ambientes de banco de dados. Os alunos têm contato direto com SGBDs como MySQL ou PostgreSQL para desenvolver os comandos aprendidos, criando procedures, functions, views e aplicando controle de acesso com DCL. São realizadas atividades práticas individuais e em grupo, com projetos simples e estudos de caso que simulam situações reais do mercado. A avaliação inclui exercícios práticos, testes teóricos e análise de código SQL produzido durante as aulas.

Aprendizados

Até o momento, a disciplina proporcionou o entendimento da estrutura de um banco de dados relacional, bem como a criação de elementos reutilizáveis como procedures e funções, úteis para automatizar operações no banco. Também foi possível aprender a construir views, que ajudam a simplificar e organizar consultas complexas, e a aplicar comandos de DCL para controlar o acesso aos dados de forma segura. Além disso, os exercícios reforçaram a lógica de programação aplicada ao SQL, contribuindo para uma visão mais ampla e segura no desenvolvimento de sistemas que utilizam banco de dados.
